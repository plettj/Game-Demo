Day 2: Once everyone's images can draw, we're gonna try to make some extremely basic javascript animation of whatever image tileset they make using Pixilart.com. Today, we decide which kind of game each person will be doing, since that will affect what we learn next: Objects (And if we are insanely productive: Physics).

Day 3: Explain OBJECTS and show them console.log() so they can test out random things to get to know objects.

Day 4: Use objects as actual entities in the game; make an avatar object with methods that make him able to be animated, and move according to key events.

Day 5: Teaching method has been revised: everyone will make a simple top-down "puzzle" game that has an avatar object along with a map object that holds the information for each level, and can draw that information to the canvas. Goals for the day include 4-way controls, and if time, map object, and if more time, 2D array to hold each level.

Day 6: Created a map object with a 2D array that can hold each level and various methods for adding levels and starting levels. Revised avatar controls to run unit-by-unit, snapping to grid and listening properly to keyboard events.
